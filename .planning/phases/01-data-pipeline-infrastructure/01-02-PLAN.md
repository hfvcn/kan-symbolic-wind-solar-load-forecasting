<task type="auto">
  <name>Task 1: Implement missing value interpolation with quality logging</name>
  <files>src/data/preprocess.py</files>
  <action>
Create `src/data/preprocess.py` with interpolation and data quality logging per user decisions:

1. `find_missing_gaps(df: pd.DataFrame, resolution_minutes: int = 5) -> list[dict]`:
   - Scan each column for consecutive NaN blocks
   - For each gap, record: column, start_time, end_time, gap_size (steps), gap_hours
   - Return list of gap dictionaries for logging

2. `interpolate_missing(df: pd.DataFrame, max_gap_hours: float = 1.0, resolution_minutes: int = 5) -> tuple[pd.DataFrame, dict]`:
   - Calculate max_consecutive = int(max_gap_hours * 60 / resolution_minutes) = 12 steps for 1 hour
   - Find all gaps using find_missing_gaps
   - Use pandas `interpolate(method='spline', order=3, limit=max_consecutive, limit_direction='forward')`
   - For gaps > max_consecutive, leave as NaN (will be handled by caller per user decision for >6 hour gaps)
   - Return (interpolated_df, gap_log_dict)

3. `handle_long_gaps(df: pd.DataFrame, gap_log: dict, strategy: str = "drop") -> pd.DataFrame`:
   - Strategy options: "drop" (remove rows), "neighbor" (use nearest valid), "keep_nan" (leave for later)
   - Default: "drop" rows with remaining NaN after interpolation
   - Log which rows were dropped

4. `generate_quality_report(df_original: pd.DataFrame, df_processed: pd.DataFrame, gap_log: dict, output_path: str) -> str`:
   - Generate detailed JSON report with:
     - Total missing values per column (before/after)
     - List of all gaps with treatment applied
     - Outlier markers (values > 3 std from mean, marked but NOT modified per user decision)
     - Summary statistics
   - Write to output_path
   - Return path for logging

5. `validate_no_nan_inf(df: pd.DataFrame) -> bool`:
   - Check for any remaining NaN or Inf values
   - Raise ValueError with details if found
   - Return True if clean

All functions should use type hints and include detailed docstrings for thesis documentation.

<task type="auto">
  <name>Task 2: Implement feature engineering (cyclic, solar, autoregressive)</name>
  <files>src/data/features.py</files>
  <action>
Create `src/data/features.py` implementing all feature groups per research patterns:

1. `encode_cyclic_features(df: pd.DataFrame) -> pd.DataFrame`:
   - Assume df.index is DatetimeIndex
   - Add hour_sin, hour_cos (period=24)
   - Add dow_sin, dow_cos (period=7 for day of week)
   - Add month_sin, month_cos (period=12)
   - Formula: sin(2*pi*value/period), cos(2*pi*value/period)
   - Return df with new columns added

2. `add_solar_features(df: pd.DataFrame, latitude: float = None, longitude: float = None) -> pd.DataFrame`:
   - If lat/lon not provided, import from src.config (ERCOT_LATITUDE, ERCOT_LONGITUDE)
   - Use pvlib.location.Location and get_solarposition()
   - Add solar_altitude (elevation in degrees)
   - Add solar_azimuth (degrees east of north)
   - Add is_night flag (solar_altitude < 0)
   - Return df with new columns

3. `add_lag_features(df: pd.DataFrame, target_cols: list[str], lags: list[int] = None) -> pd.DataFrame`:
   - Default lags = list(range(1, 49)) for t-1 to t-48 per user decision
   - For each target_col, create {col}_lag_{n} using df[col].shift(n)
   - DO NOT drop rows here (caller handles after feature engineering)
   - Return df with lag columns

4. `add_all_features(df: pd.DataFrame, target_cols: list[str], latitude: float = None, longitude: float = None) -> pd.DataFrame`:
   - Convenience function combining all feature types
   - Call encode_cyclic_features, add_solar_features, add_lag_features in sequence
   - Return fully-featured df

5. `get_feature_groups() -> dict[str, list[str]]`:
   - Return dict mapping feature group names to column name patterns:
     - "cyclic": ["hour_sin", "hour_cos", "dow_sin", ...]
     - "solar": ["solar_altitude", "solar_azimuth", "is_night"]
     - "lag": pattern "{col}_lag_{n}"
   - Useful for feature selection and documentation

Note: Lag features create NaN in first 48 rows. This is handled in split.py (Plan 01-03) after splitting to avoid data leakage.
