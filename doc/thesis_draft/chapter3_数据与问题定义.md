# 第三章 数据与问题定义

## 3.1 数据来源与基本描述

本研究采用美国能源部高级研究计划局（ARPA-E）发布的PERFORM公开数据集[33]，选取德克萨斯州电力可靠性委员会（ERCOT）区域2018年全年数据作为实验对象。该数据集以5分钟为原始采样间隔（即每日288个时间步），包含负荷（load）、风力发电（wind generation）与光伏发电（solar generation）三类平衡区（BA）级别的实测序列。2018年全年共约105,120条记录。数据以HDF5格式存储于AWS S3公开存储桶，通过匿名访问获取[33]。

选择ERCOT区域的理由有三：第一，ERCOT作为美国最大的独立系统运营商之一，风光渗透率居全美前列，其净负荷波动包含显著的可再生能源不确定性[2]；第二，PERFORM数据集对ERCOT区域提供了同期气象重分析变量，使得气象物理量与发电量之间的因果路径可被追踪；第三，ERCOT系统的电力市场独立性使得跨区域潮流干扰较小，有利于控制实验变量。

## 3.2 净负荷与预测目标设计

### 3.2.1 净负荷定义

净负荷（net load）是衡量电力系统中常规可调度机组实际承担的供需缺口的核心指标，其定义为：

$$\text{net\_load}(t) = \text{load}(t) - \text{wind}(t) - \text{solar}(t)$$

净负荷综合了用电需求与可再生能源出力两方面的不确定性。相比分别预测负荷和发电量再做差值，直接预测净负荷能够捕捉三者之间的相关结构；但同时也带来了可解释性挑战——当模型以净负荷为目标时，各物理驱动因素（温度驱动负荷、风速驱动风电、辐照驱动光伏）的贡献路径在目标层面发生了混叠。

### 3.2.2 差分目标构造

本研究以差分形式（delta target）作为预测目标，而非直接预测净负荷的绝对值。对于预测时域$h$（单位：时间步，1步=5分钟），差分目标定义为：

$$\Delta\text{net\_load}_{h}(t) = \text{net\_load}(t+h) - \text{net\_load}(t)$$

采用差分目标的考虑如下。第一，差分序列消除了净负荷的缓变趋势和日内基线，使模型聚焦于短期变化量的学习，这与调度运行中关注"未来半小时负荷变化多少"的实际需求直接对应。第二，差分目标天然具有零均值特性，有利于KAN（Kolmogorov-Arnold Network）的B样条激活函数在训练初期快速收敛。第三，差分预测的精度可通过持久性模型（persistence model，即预测$\Delta = 0$）给出自然基线，由此定义技巧分数（Skill Score），便于量化模型相对于朴素策略的增益。

本研究设置三个预测时域：

- $h=6$：对应30分钟提前量，主要目标，用于全部实验阶段
- $h=12$：对应1小时提前量，用于S1多时域对比
- $h=24$：对应2小时提前量，用于S1多时域对比

相应地，主实验的目标变量为$\text{delta\_net\_load\_h6}$。在结构化分解实验（S3，详见第四章4.6节）中，额外定义分量差分目标：

$$\Delta\text{load}_{h6}(t) = \text{load}(t+6) - \text{load}(t)$$

$$\Delta\text{wind}_{h6}(t) = \text{wind}(t+6) - \text{wind}(t)$$

$$\Delta\text{solar}_{h6}(t) = \text{solar}(t+6) - \text{solar}(t)$$

这三个分量目标与净负荷目标之间满足线性恒等式：$\Delta\text{net\_load}_{h6} = \Delta\text{load}_{h6} - \Delta\text{wind}_{h6} - \Delta\text{solar}_{h6}$，这一性质将在S3实验中被用于组合预测。

## 3.3 特征工程

本研究的特征体系分为四大类：周期编码特征、太阳几何特征、气象物理特征和自回归滞后特征。特征设计遵循两条原则：一是尽可能注入领域物理知识（如风功率正比于风速立方、光伏效率随温度线性衰减），二是保持特征的可解释性以便在符号回归阶段追踪变量来源。

### 3.3.1 周期编码特征

时间序列的周期性模式通过正弦-余弦（sin/cos）编码捕捉，避免线性编码在周期边界处产生的不连续跳变[34]。编码公式为：

$$x_{\sin} = \sin\left(\frac{2\pi \cdot v}{P}\right), \quad x_{\cos} = \cos\left(\frac{2\pi \cdot v}{P}\right)$$

其中$v$为时间值，$P$为周期长度。具体编码如下：

- **小时编码**（$P=24$）：$v = \text{hour} + \text{minute}/60$，输出hour\_sin、hour\_cos，捕捉日内负荷波动
- **星期编码**（$P=7$）：$v = \text{dayofweek}$（周一=0），输出dow\_sin、dow\_cos，捕捉工作日/周末效应
- **年周期编码**（$P=365.25$）：$v = \text{dayofyear}$，输出month\_sin、month\_cos，捕捉季节性变化

周期编码共产生6个特征，构成cyclic特征组。

### 3.3.2 太阳几何特征

太阳位置特征基于pvlib天文计算库[35]，以ERCOT区域质心坐标（纬度31.0°N，经度100.0°W）为参考点计算：

- **太阳高度角**（solar\_altitude）：太阳相对于地平面的仰角，单位为度，范围$[-90°, 90°]$
- **太阳方位角**（solar\_azimuth）：太阳相对于正北方向的方位角，单位为度，范围$[0°, 360°]$
- **夜间标志**（is\_night）：布尔变量，当solar\_altitude $< 0$时为真

太阳几何特征在细粒度分析中进一步分为solar\_geom组（solar\_altitude、solar\_azimuth）和solar\_flag组（is\_night），以支持消融实验中对辐照信息源（天文几何 vs. 地面实测GHI）的独立控制。

### 3.3.3 气象物理特征

气象特征来自PERFORM数据集附带的气象重分析数据，并经过领域知识引导的工程化处理。按物理含义分为以下子组：

**风速类特征（meteo\_wind组）：**

- wind\_speed\_10m\_m\_s：10米高度风速原始值，单位m/s
- wind\_speed\_10m\_m\_s\_cubed：风速立方值$v^3$，作为风功率物理代理。根据贝兹极限（Betz limit），风力发电机理论最大可提取功率正比于来流动能通量$P \propto \frac{1}{2}\rho A v^3$[36]，因此$v^3$比线性风速更直接地关联风力发电出力
- wind\_speed\_hub\_est：轮毂高度风速估计值，通过幂律风廓线外推得到：

$$v_{\text{hub}} = v_{10\text{m}} \times \left(\frac{h_{\text{hub}}}{h_{\text{meas}}}\right)^{\alpha}$$

其中$h_{\text{hub}} = 100\text{m}$为典型ERCOT风电场轮毂高度，$h_{\text{meas}} = 10\text{m}$为气象站测量高度，$\alpha = 0.14$为开阔地形标准Hellman指数[37]。该特征较10米风速与实际ERCOT风力发电出力具有更高的相关性。

**辐照类特征（meteo\_irradiance组）：**

- ghi\_w\_m2：全球水平辐照度（Global Horizontal Irradiance, GHI），单位W/m²
- ghi\_day\_w\_m2：日间门控辐照度，定义为$\text{ghi\_day} = \text{ghi} \times (1 - \text{is\_night})$，消除夜间GHI传感器噪声
- ghi\_temp\_corr\_w\_m2：温度修正辐照度，考虑晶硅光伏组件效率随温度的线性衰减[38]：

$$\text{ghi\_corr} = \text{ghi} \times \left[1 - \gamma \cdot (T - T_{\text{ref}})\right] \times (1 - \text{is\_night})$$

其中温度系数$\gamma = 0.004\text{ /°C}$，参考温度$T_{\text{ref}} = 25\text{°C}$。该特征同时编码了辐照强度和温度对光伏出力的耦合效应。

**温度类特征（meteo\_temp组）：**

- temp\_2m\_c：2米高度气温，单位°C，是负荷需求的核心气象驱动因素

**度日特征（meteo\_degree组）：**

- cdd\_18c：制冷度值（Cooling Degree），$\text{CDD} = \max(0, T - 18\text{°C})$，反映制冷需求压力
- hdd\_18c：制热度值（Heating Degree），$\text{HDD} = \max(0, 18\text{°C} - T)$，反映供暖需求压力

度日特征将气温的双向非线性效应（高温增加空调负荷、低温增加供暖负荷）转化为单调递增的代理变量，在负荷预测研究中应用广泛[39]。在5分钟分辨率下，这些是瞬时代理值而非传统日累积量。

**气压类特征（meteo\_pressure组）：**

- surface\_pressure\_hpa：地表气压，单位hPa，间接反映天气系统变化

### 3.3.4 自回归滞后特征

自回归（autoregressive）滞后特征通过对负荷、风力发电和光伏发电三个序列施加时间平移得到。本研究选取三个代表性滞后步数：

- lag\_12（1小时回溯）：捕捉短期惯性
- lag\_24（2小时回溯）：捕捉中期趋势
- lag\_48（4小时回溯）：捕捉较长期动态

滞后特征以$\{\text{series}\}\_\text{lag}\_\{n\}$命名，如load\_lag\_12表示负荷序列向前平移12步（即$\text{load}(t-12)$）。三个序列各取三个滞后步数，共产生$3 \times 3 = 9$个滞后特征。

多尺度滞后的设计参考了MCKAN[40]中多尺度卷积核的思想：不同滞后步数相当于在不同时间尺度上观察序列惯性，模型可以通过选择性激活不同滞后特征来自适应地确定最优回溯窗口。

滞后特征在本研究中承担了双重角色。一方面，它们为模型提供了强预测信号——由于时间序列的强自相关性，滞后值往往是最佳的短期预测器。另一方面，正是这种预测优势导致了本研究的核心发现：在KAN的L1+熵稀疏化过程中，自回归滞后特征因其捷径竞争优势（shortcut competition），将气象物理量的激活边压制为零，从而使后者在剪枝阶段被系统性剔除。这一现象构成了本文第五章实验分析的核心观察。

## 3.4 数据划分与标准化

### 3.4.1 时序划分策略

时间序列数据的划分必须遵循严格的时序一致性原则，禁止随机打乱。本研究采用按时间顺序的60/20/20（训练/验证/测试）三段式划分，并在相邻子集之间设置48步（4小时）的间隔缓冲区（gap），以防止滞后特征造成的信息泄漏。

以2018年全年数据（约105,120条记录）为例，划分结果大致为：

- **训练集**：2018年1月1日至约8月中旬，约63,072条
- **验证集**：约8月中旬至约10月下旬（前置48步间隔），约21,024条
- **测试集**：约10月下旬至12月31日（前置48步间隔），约20,928条

间隔缓冲区的步数设为48，与最大滞后步数（lag\_48）一致，确保验证集的首条记录不包含任何训练集中的观测值作为滞后输入。

### 3.4.2 Z-score标准化

特征标准化采用Z-score方法，仅在训练集上拟合均值和标准差参数：

$$x_{\text{scaled}} = \frac{x - \mu_{\text{train}}}{\sigma_{\text{train}}}$$

该参数随后应用于验证集和测试集的变换，避免标准化过程中的未来信息泄漏。标准化参数以JSON格式持久化保存，供模型推理和符号公式逆变换使用。

### 3.4.3 已知局限性

数据预处理流程中存在一项已知的技术缺陷：线性插值（用于填补原始数据中的少量缺失值）在训练/验证/测试划分之前执行，而非在划分后分别对每个子集独立插值。这意味着划分边界附近的插值结果可能使用了相邻子集的信息，构成轻微的信息泄漏。然而，该泄漏具有以下特点：(1) 仅影响划分边界附近极少数样本；(2) 是全局性的数据处理步骤，不针对特定变量或特征组；(3) 对本研究关注的变量间相对可辨识性差异不产生系统性偏倚。因此，该局限性被文档记录但不影响实验结论的有效性。

## 3.5 评估指标体系

本研究采用以下指标评估模型预测性能：

**均方根误差（Root Mean Square Error, RMSE）：**

$$\text{RMSE} = \sqrt{\frac{1}{N}\sum_{i=1}^{N}\left(y_i - \hat{y}_i\right)^2}$$

RMSE对大误差赋予更高惩罚，反映模型在极端偏差场景下的表现。

**平均绝对误差（Mean Absolute Error, MAE）：**

$$\text{MAE} = \frac{1}{N}\sum_{i=1}^{N}\left|y_i - \hat{y}_i\right|$$

MAE提供误差尺度的直观度量，对离群值的敏感度低于RMSE。

**决定系数（Coefficient of Determination, $R^2$）：**

$$R^2 = 1 - \frac{\sum_{i=1}^{N}(y_i - \hat{y}_i)^2}{\sum_{i=1}^{N}(y_i - \bar{y})^2}$$

$R^2$衡量模型解释目标变量方差的比例，取值范围为$(-\infty, 1]$，$R^2 = 1$表示完美拟合。

**技巧分数（Skill Score）：**

$$\text{Skill} = 1 - \frac{\text{RMSE}_{\text{model}}}{\text{RMSE}_{\text{persistence}}}$$

其中持久性模型以当前时刻值作为预测值（即$\hat{\Delta} = 0$）。技巧分数为正表示模型优于持久性基线；其值越大，表明模型相对于朴素策略的增益越高。

**统计显著性检验**：对于需要比较两组预测结果的实验场景（如阻断实验的成对比较），采用配对t检验（paired t-test）判断差异的统计显著性。

## 3.6 问题形式化

综合以上数据设计与实验背景，本研究的核心问题可形式化表述如下。

**给定**：ERCOT 2018年5分钟分辨率的负荷、风力发电、光伏发电序列及同期气象重分析变量，构造特征集$\mathbf{X} = [\mathbf{x}_{\text{cyclic}}, \mathbf{x}_{\text{solar}}, \mathbf{x}_{\text{meteo}}, \mathbf{x}_{\text{lag}}]$和差分目标$\Delta\text{net\_load}_{h}$。

**训练**：以KAN网络为教师模型，经L1+熵正则化稀疏训练和结构化剪枝后，提取闭式符号公式$f_{\text{sym}}(\mathbf{X})$。

**观察**：在标准训练流程下，气象物理变量（如wind\_speed\_hub\_est、ghi\_temp\_corr\_w\_m2等）在稀疏化过程中被系统性剪除，未能进入最终符号公式。

**核心问题**：该现象的主要机制是什么？是气象变量对目标的预测贡献确实可忽略（变量无关性假说），还是自回归滞后特征的捷径竞争在稀疏化过程中挤出了气象变量（捷径竞争假说）？

**研究策略**：通过控制变量实验——阻断自回归滞后特征的输入通路——测量气象变量可辨识性的变化，从而区分上述两种机制。若阻断滞后特征后气象变量的可辨识性显著提升，则支持捷径竞争假说；若无变化，则支持变量无关性假说。

## 3.7 本章小结

本章介绍了研究所用的ARPA-E PERFORM数据集，详细描述了净负荷差分目标的构造逻辑和多时域设计。在特征工程方面，按照物理含义将28个输入特征组织为周期编码、太阳几何、气象物理和自回归滞后四个大类，并阐述了风速立方、轮毂风速估计、温度修正辐照度等物理代理特征的设计依据。数据以60/20/20时序划分并进行Z-score标准化，评估采用RMSE、MAE、$R^2$和技巧分数。最后，将研究核心问题形式化为自回归捷径竞争假说与变量无关性假说之间的判别问题，为下一章的方法设计提供了明确的实验目标。
