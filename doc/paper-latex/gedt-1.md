## Part 1 [The Review Report]

**Summary：** 文章主张在高频净负荷预测中，KAN 符号提取之所以退化为只含滞后项的公式，主要不是因为气象变量“没有信息”，而是因为自回归捷径在稀疏化阶段压制了物理变量；作者进一步用阻断实验与 S3 结构化分解给出机制证据和补救方向。

**Strengths：**

1. 这不是一篇泛泛的“把 KAN 用到能源预测”的应用文，而是抓住了一个确实有研究价值的 failure mode：时序任务中的自回归特征会系统性破坏符号可辨识性。这个问题如果被坐实，对 KAN-to-symbol、乃至更广泛的时序符号回归社区都具有外溢价值。

2. 实验链条有明确的问题导向，而不是堆表格。稿件先确认 direct 路线的公式坍缩，再用 solar 正例和 wind 边界案例做辨析，然后用 blocking 进行机制检验，最后给出 S3 分解方案。VER/FAR/TGR 虽然不复杂，但确实把“变量是否真正进入公式”从口头描述变成了可追踪对象，这一点有方法学意义。

3. S3 至少展示出一个值得社区继续追的方向：把净负荷拆成负荷、风电、光伏子任务后，局部公式中的目标物理量可以稳定出现，而组合预测精度与 direct KAN 基本持平。即便当前还不是闭环的总公式，这个方向本身是有价值的。

**Weaknesses (Critical)：**

1. 最关键的 Case 4 证据链并不是严格的 apples-to-apples 对照。未阻断端使用的是 9 组 direct symbolic 配置上的最终公式统计，阻断端使用的是 5 个 seed 的“活跃边”统计；两边的分母不同、统计对象不同、层级不同，而且 RMSE 口径也从主表的 abs(test) teacher 评估变成了 blocked/pruned-delta 评估。于是文中最重要的 (\Delta)VER=+0.80 并不是同一统计量上的严谨前后对照，它只能算强烈暗示，不能算已经闭合的机制证据。对于一篇把“主导机制”作为核心主张的论文，这是最伤评分的一点。

2. 论文没有真正排除“条件冗余”这个替代解释。作者证明了物理量在 focused task 中、或在移除 lag 后，可以重新进入公式；但这并不等价于证明：在 direct net-load + full lags 的原任务里，这些气象量本来就应当被保留。当前 lag-only 公式也可能是一个稀疏但合理的条件最优解，而不一定是作者定义下的“坍缩”。缺少 direct task 上的 lags-only vs lags+meteo、shuffled-meteo、matched-dimension control，使“不是变量无关性，而是捷径竞争”这一概念判断仍然有逻辑空缺。

3. S3 的结论写得比证据更强。现在保住精度的是 structured composite predictor，拿到可解释性的是三个局部公式；二者并不是同一个 artifact。论文自己也承认尚未显式合成净负荷总公式并在 test 上逐点复算。更关键的是，表中的 TGR 为 1.401–2.569，说明局部 symbolic formula 对 teacher 的保真度并不高。因此目前最多能成立的结论是“分解后可恢复局部物理公式”，而不是“已经实现端到端的高保真可解释净负荷预测”。

4. 按顶会标准，实证广度和统计严谨性仍偏弱。全文只在单一区域、单一年份上验证，作者还明示了划分前插值带来的泄漏；主精度显著性是基于强时序相关的逐点误差，而不是 seed-level repeated runs 或 block-aware 统计；缺少若干关键控制 baseline，例如 direct task 下的 lags-only 线性/非线性模型，以及更公平的 non-KAN 稀疏/符号对照。此外还有几处不致命但会伤害可信度的表述问题，例如特征数 26/28 不一致、前文主时域写 h=6/12/24 而后文又大量使用 72/144/576、S3 局部公式正文未给出。这些不是致命问题，但会进一步压低评审信心。

**Rating：** 总体 **5/10**。这篇稿子的问题意识是真实的，贡献也不是虚的，但核心机制 claim 还没有被严格钉死，S3 解决方案也尚未形成同一件可验证的符号产物；按 **ICLR/NeurIPS/ICML** 标准我会给 **弱拒（约 4–5/10）**，按 **AAAI/IJCAI/KDD** 标准大致在边界线附近，但仍 **略低于录用（约 5–6/10）**。

## Part 2 [Strategic Advice]

1. **关于 Case 4 对照不一致。**
   问题根源不是方法本身，而是实验 protocol 没有把“阻断前后”做成同一统计口径下的配对比较：一边看最终公式，一边看活跃边；一边是 9 组 symbolic config，一边是 5 个 seed；一边是 teacher 精度，一边是 pruned-delta 精度。这个问题**可救性高**，但必须靠补实验而不是润色。最直接的做法是：固定同一组 seed，同一 symbolic grid，同一评估口径，在 blocked/unblocked 上同时报告 VER、FAR、TGR 和最终公式精度；再给出逐 seed 的 paired plot 或 block-bootstrap CI。若 rebuttal 来不及重跑，必须主动把“主导机制已确认”降级为“获得支持性证据”，否则很容易被 reviewer 抓住统计口径不一致这一点直接否掉主张。

2. **关于“捷径竞争”与“条件冗余”混淆。**
   这里的深层问题是概念层面的：你现在证明的是“去掉 lag 后，天气变量会回来”，但没有证明“在原始任务里，天气变量本来就不该被剪掉”。这二者不是一回事。这个问题**可救性中等**；如果 direct task 中天气变量对增量精度、极端天气日、OOD 泛化确实有独立贡献，那么可以靠补实验把逻辑补上；如果没有，那么就不是实验没做够，而是主叙事本身需要收缩。建议至少补四组 direct net-load 对照：lags-only、meteo-only、lags+meteo、lags+meteo 但将 meteo 打乱；最好再加一个 matched-dimension control，例如移除等数量的非 lag 特征，而不是只移除 lag。只要这些实验一做，评审就能分清：你看到的是“自回归捷径竞争”，还是“lag 已经充分总结了天气信息，因此稀疏公式合理地丢掉了外生变量”。如果结果并不支持后者，就把主张改写成更稳妥的版本：不是“排除了变量无关性”，而是“现有证据更支持 shortcut competition 与 conditional redundancy 的共同作用，其中前者在当前管线里可观测”。

3. **关于 S3 贡献边界与 TGR 偏高。**
   问题根源是你现在把“预测器的精度”和“公式的可解释性”绑在一起写，但这两者并不来自同一个最终对象。评审会非常敏感地指出：你保住精度的是 composite predictor，不是 composite symbolic formula；而且局部公式的 TGR 明显不低，说明 fidelity 还不够强。这个问题**中高可救**。最优解是尽快显式构造 (F_{\text{net}}=F_{\text{load}}-F_{\text{wind}}-F_{\text{solar}})，然后在 test 上逐点复算，单独给出 composite symbolic 的 RMSE/MAE/Skill/TGR。与此同时，把三条局部公式至少各展示一条代表式，别只给 FAR 和边数。若显式合成后的总公式也能接近 composite predictor，那 S3 会从“有潜力的方向”直接升格为真正的贡献点；若做不到，也不要硬撑，把论文定位收紧到“恢复局部物理公式的结构化方案”，这仍然是一个成立的贡献，只是比现在的写法窄一些。

4. **关于广度、统计与可复现性。**
   这些问题的根源是稿子目前更像一篇“研究收敛过程的毕业论文”，还没有完全切换到“顶会可审计 protocol”的写法。这里面有些是**修订期可解决**的，例如补 optimizer/训练细节、明确 bootstrap 方式、统一 horizon 叙事、修正 26/28 特征数矛盾、把局部公式放进正文、把逐点 permutation 检验换成更合理的 seed-level 或 block-aware 统计；但也有些是**接近结构性缺陷**的，例如划分前插值导致的泄漏、单一数据集、单一区域，这些往往需要真实重跑，rebuttal 期很难彻底补齐。我的建议是：若目标是 ICLR/NeurIPS/ICML，这一层面基本都要补；若目标是 AAAI/IJCAI/KDD，至少要把泄漏问题写清、把统计改正、再补一个外部验证场景或第二个区域，否则“机制性结论”很难站稳。

如果只能做最少量、但最提分的修改，优先级应当是：先把 **Case 4 做成同口径对照**，再补 **direct task 上区分 shortcut competition 与 conditional redundancy 的控制实验**，然后尽快给出 **S3 显式总公式及其 test 复算**。这三步一旦做实，论文会从“有趣但证据不闭环”明显抬升到“有明确新发现且证据链基本完整”。
