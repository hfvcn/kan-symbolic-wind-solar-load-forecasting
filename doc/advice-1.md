下面给出一个“能直接落到你现有 Phase1–8 流水线 + `scripts/experiment_driver.py` 调参入口”的改进方案，目标是：**在不作弊、不靠平凡解的前提下**，让最终可解释公式里**同时出现历史负荷 + 温度/光照(GHI)/风速（或其代理）**，并把 test 集 **R² 从长期负数拉回可用区间**。方案完全基于你交接文档里描述的现状与接口约束来设计。

## 0. 先把“为什么现在会卡死”说清楚（论文也需要这段）

你现在遇到的现象在 5min ahead 负荷预测里是“正常但致命”的：

* **`load_lag_1` 近似等价于“持久性预测”**（persistence）：负荷序列在 5 分钟尺度上极连续，所以最优策略往往就是复制上一时刻。于是

  * 精度极高（RMSE 很小、R² 很高），
  * 但外生物理因子（温度/GHI/风速）在边稀疏化 + 符号拟合时会被当作“微小修正项”剪掉或忽略。
* 你已经在实验里验证了两端极限：

  * **含 `load_lag_1`：性能超强但公式几乎纯自回归**；
  * **去掉所有 lag：物理量进公式但 R² 变负**。

所以“兼顾性能 + 公式含物理因子”的关键，不是再硬调一遍同一个设置，而是要**改变任务的可学习结构**：让“历史负荷负责主趋势”，同时让“物理因子负责可解释的修正/耦合项”，并且这种修正项要在指标上真实贡献（否则还是会被稀疏化淘汰）。

---

## 1. 总体策略：用“分解式建模”破局（推荐作为主线）

把目标写成一个论文友好、工程也合理的形式：

[
\hat{L}*{t+1}= \underbrace{L_t}*{\text{持久性/惯性}} ;+; \underbrace{f(\text{temp},\text{GHI},\text{wind},\text{solar_pos},\text{cycle},\text{(可选)慢滞后})}_{\text{可解释的物理与周期修正}}
]

也就是：**预测“负荷变化量”而不是直接预测“负荷绝对值”**。

* 定义新目标：
  [
  \Delta L_{t+1}=L_{t+1}-L_t
  ]
* 用 KAN→符号提取去拟合 (\Delta L)（它的量级更小、更接近“由外生驱动造成的变化”），并强制/引导模型把温度、GHI、风速纳入关系式。
* 最终恢复到负荷预测：
  [
  \hat{L}*{t+1}=L_t+\widehat{\Delta L*{t+1}}
  ]

**这条路线的优点：**

1. 你不用和 `load_lag_1` 正面硬刚：它直接被放到等式左边当作“已知基线”，不会在符号化时吞掉一切；
2. 公式天然包含“历史负荷”（因为右侧第一项就是 (L_t)，这在论文叙事上非常干净）；
3. 右侧的 (f(\cdot)) 很容易在符号上体现温度/辐照/风速的贡献，也更像“潜在物理规律/关键因素”。

> 这也是你在文档里提到的“性能与可解释性折中”的最自然方式：把强自回归当成“可解释的结构先验”，再在残差里挖物理。

---

## 2. 让公式更“物理化”的小技巧：做 3 个可解释特征变换（强烈建议）

你现在的符号库里有 `abs`、多项式、sin/cos 等。利用这些，可以很自然地构造“电力负荷领域常用”的物理量：

### 2.1 冷/热负荷代理（Cooling/Heating Degree）

选一个基准温度 (T_0)（比如 18°C 或 20°C，论文里说明为“舒适温度阈值”），构造：

* 冷负荷代理：(CDH=\max(0, T-T_0))
* 热负荷代理：(HDH=\max(0, T_0-T))

如果你不想在 Phase1 里加新列，也可以靠 `abs` 在符号里表达：
[
\max(0, x)=\frac{x+|x|}{2}
]
所以 CDH/HDH 都能被符号回归学出来；但**更推荐 Phase1 直接生成特征列**，这样更稳定、更易进入公式。

### 2.2 风功率代理：(wind_speed^3)

风机功率对风速约 (\propto v^3)（低风速段尤其明显）。你库里本来就有 (x^3)，但把它做成显式特征（如 `wind_speed_10m_m_s_cubed`）会更容易让公式出现“风影响”且更符合物理叙事。

### 2.3 光伏代理：夜间门控 + 太阳高度

你已有 `is_night`、`solar_altitude`、`ghi_w_m2`。非常建议引导模型形成类似：

* (PV_proxy \approx GHI \cdot \max(0,\sin(\text{solar_altitude})))
* 或简化为：白天 `ghi` 起作用，夜间为 0（门控）。

同样：Phase1 显式生成 `ghi_day = ghi_w_m2 * (1 - is_night)` 会让符号提取更稳定。

> 这些变换的意义在论文里很好写：把“可解释性”提升到“贴近工程物理机理”的层面，而不是纯数学拟合。

## 3. 四组“可直接用 experiment_driver 做 sweep”的实验方案（2–4 组要求满足）

下面给你 **4 组 sweep**。你可以只跑前 2 组就能明显改善论文叙事；后 2 组用于“耦合负荷”主题加分。

我会把每组的目标、特征选择、KAN 训练倾向、符号提取约束、以及你该看什么指标写清楚。

---

### Sweep A（主线，优先跑）：ΔLoad 残差建模，让物理项必然出现

**目标**：让最终公式结构变成
[
\hat{L}_{t+1}=L_t + f(\text{temp}, \text{GHI}, \text{wind}, \text{cycle}, \text{solar_pos}, \text{(可选)慢滞后})
]
并在 test 上得到**非负、可用的 R²**，同时公式里明确出现 temp/GHI/wind。

**实现要点（Phase1）**
在数据管道里新增一列：`delta_load = load - load_lag_1`（或者如果你是预测 t+1，就按你训练口径做 shift 后差分）。然后把 `target` 从 `load` 改为 `delta_load` 跑一个新的 data_run。

> 这一步改动非常小，但价值巨大：它把“惯性”剥离掉，把学习任务变成“变化由什么导致”。

**特征选择建议（用你现有 CLI 能做到）**

* `include_groups`: `meteo + solar_pos + cycles + lags`
* `lag_series`: 建议 **只保留 wind/solar 的 lag + 少量慢的 load lag**，例如：

  * load 只保留 `lag_12, lag_24, lag_48`（代表 1h/2h/4h 的“慢记忆”）
  * wind/solar 可以保留 `lag_1..48` 或者也只留 `1,12,24,48`（看你计算量）
* 显式剔除 `load_lag_1`（因为它已经用于 delta 的定义，继续给模型会形成退化解）

**KAN 训练建议（针对你现在“KAN 明显弱于简单基线”的症状）**

* 不要一上来就把稀疏正则压到极强。先保证 KAN 在 ΔLoad 上能拟合出合理 RMSE，再逐步加 sparsify。
* sweep 两档正则强度：

  * A1：弱稀疏（更重性能）
  * A2：中稀疏（更重可解释）

**符号提取建议**

* `--lib`：先从“更可解释”的库开始，避免 exp/gaussian 把一切吸走：

  * `x,x^2,x^3,sin,cos,abs`
* `r2_threshold`：建议 0.97–0.995 做一维 sweep（阈值越高越“可信”，但可能导致某些边无法符号化）

**你要看的评估指标（本地 Phase5–8）**

1. 还原后的负荷指标：`rmse(load) / r2(load)`
2. 物理映射：至少 `temp`、`ghi`、`wind` 的敏感性存在（导数/散点）
3. 复杂度：node_count/tree_depth 与 RMSE 的 Pareto
4. 最关键：**相对持久性基线的 skill score**

   * persistence：(\hat L_{t+1}=L_t)
   * 你的模型：(L_t + f(\cdot))
   * 报告“RMSE 相对下降百分比”，这在 5min 任务里比单纯 R² 更能说明“模型学到了什么”。

> Sweep A 一旦跑通，你的论文核心就稳了：既解释了为何 5min 预测容易退化，又给出了解法，并且产出一个物理友好的解析修正项。

---

### Sweep B（不改 target 的备选）：限制滞后集合，迫使模型用外生物理量

**目标**：仍然预测 `load`，但通过 **特征约束** 把 `load_lag_1` 的统治力削弱，让外生项进入公式，同时尽量不让 R² 崩到负数。

**特征选择**

* 关键：**保留 load 的“慢滞后”，剔除“快滞后”**

  * load：只留 `lag_12, lag_24, lag_48`（甚至再加 `lag_36`）
  * wind/solar：可留 `lag_1..48`（它们本身是耦合因素）
  * meteo/solar_pos/cycles 全保留
* 这等价于在论文里说：“为了避免 5min 持久性带来的平凡解，我们刻意采用更符合调度尺度的滞后特征集。”

**KAN + 符号提取**

* 同 Sweep A 的符号库约束（先禁 exp/gaussian）
* 稀疏强度可以比 Sweep A 稍强，因为这里输入维度可能更大

**预期**

* 性能会比允许 `load_lag_1` 差（这是必然），但有机会从负数拉回到正的 R²，且公式更容易出现外生项。
* 这组可以作为“折中对照”：证明不是只能靠 ΔLoad 才能做到，但 ΔLoad 更优。

---

### Sweep C（论文加分项）：改预测尺度（30min / 1h ahead），让物理因子“自然重要”

**目标**：解决“5min 预测太容易退化”的根因：把预测 horizon 拉长到 30min 或 1h，让温度/GHI/风速对未来更有解释力。

**做法**

* Phase1 新增 `horizon_steps`（或你用现有方式做 target shift），例如：

  * 30min：h=6
  * 1h：h=12
* target = `load_{t+h}` 或 `net_load_{t+h}`（见 Sweep D）
* 特征仍然只能用 (t) 时刻及以前的 lag（避免泄漏）

**为什么这对你的“论文要求”更贴合**

* 电网调度通常不会只看 5min ahead；30min/1h 更常见，也更需要“可解释、可信”的模型。
* 此时 `load_lag_1` 不再近似等价于答案（它距离目标更远），外生因子更有机会进入公式且贡献更大。

**建议**

* 这组不用跑太多超参，主要是验证“尺度改变 → 可解释性显著改善”的因果链。

---

### Sweep D（耦合主题最贴合）：把目标改成 Net Load 或“耦合残差”，但避免平凡恒等式

你文档里已经点出：如果论文写“风光耦合负荷”，但 target 实际是 `load`，那风/光进入公式就不自然。这里给两种不作弊写法：

#### D1：预测 Net Load（定义层面的耦合）

定义（论文中要明确这是**定义**不是“发现”）：
[
net_load = load - wind - solar
]

**关键避免平凡解的方法**

* 训练时 target 用真实 `net_load_{t+1}`（或 `t+h`），但输入特征**不允许**包含同一时刻的 `wind_{t+1}`/`solar_{t+1}`（你现在主要是 lag 特征，本来就不会直接给到未来值）。
* 让模型只能用：

  * `load_lag_*`, `wind_lag_*`, `solar_lag_*`
  * meteo + solar_pos（这些可视为已知/可预测的外生）
* 这样学到的不是恒等式，而是“净负荷变化如何由天气与可再生出力惯性决定”。

#### D2：耦合残差（更像“发现”）

做两步分解：

1. baseline：用 `load_lag_1`（或一个很简单的 AR/MLP）预测 (load_{t+1})
2. 残差：
   [
   r_{t+1}=load_{t+1}-\hat load^{baseline}*{t+1}
   ]
   用符号回归去拟合 (r*{t+1}) 与（wind/solar lags + meteo/GHI/solar_pos）的关系。

这在论文里可以表述为：
“我们把高频惯性与可再生耦合影响分离；耦合影响主要体现在对 baseline 的修正项上，并可由解析公式刻画。”

## 4. 额外建议：把“选择最优 run”的规则从单指标改成三指标（你现有资产体系正好支持）

你本地已经生成 `comparison_table.csv`、Pareto 图、`physical_score` 等。建议把最终“论文主结果”的筛选规则明确写成：

* **硬门槛**：test R² ≥ 0（或 ≥ 0.5，按你想要的“可用性”）
* **可解释性门槛**：复杂度 ≤ 某个阈值（node_count/tree_depth）
* **物理门槛**：physical_score ≥ 1（至少温度/GHI/风速中两类能通过敏感性检查）
* 在满足门槛的集合里，再做 RMSE 最小或 Pareto 最优。

这能避免你陷入“指标最好但公式没物理项”的尴尬，也符合论文要求“可解释性同等重要”。

## 5. 建议补充的评估与图（能直接写进论文）

这些基本都能用你 Phase5–8 的本地脚本体系产出（或小改动即可）：

1. **Persistence baseline**（一定要单列）

   * (\hat L_{t+1}=L_t) 的 RMSE/R²
   * 用它解释为什么 5min 指标会“虚高”，也解释你的研究动机：不仅要准，还要理解耦合机理。

2. **Skill score / 改善率**

   * (\text{skill} = 1 - \frac{RMSE_{model}}{RMSE_{persist}})
   * 这对高频任务非常关键，比“绝对 RMSE”更能说明贡献。

3. **敏感性分布 + 物理一致性**（你已有雏形）

   * (\partial \hat y / \partial temp)、(\partial \hat y / \partial ghi)、(\partial \hat y / \partial wind) 的分布
   * 分季节/昼夜对比（夜间 GHI 的敏感性应接近 0）
   * 这会让“可解释性”更可信。

4. **残差结构**

   * 对比 baseline 残差 vs 加入符号修正后的残差：是否减少系统性偏差（例如高温时段偏差）

## 6. 论文叙事结构建议（把“高精度基线”和“物理发现”拼成可信论文）

你可以按“问题—退化—解法—发现”的逻辑写：

1. **任务与挑战**：5min 风光耦合负荷预测既要准又要可解释。
2. **基线事实**：在 5min 尺度，`load_lag_1` 导致持久性基线极强 → 任何追求 RMSE 的模型都会退化成自回归公式（你已用 PySR/MLP 证明）。
3. **方法贡献**：提出“分解式建模”（惯性项 + 物理修正项），用 KAN 的稀疏结构保证可符号化。
4. **结果**：

   * 精度：相比 persistence 的 skill 提升；
   * 可解释性：给出解析公式（LaTeX/SymPy）可复算 test 指标；
   * 物理性：温度/辐照/风速敏感性符合常识（昼夜/季节一致性）。
5. **耦合讨论**：若用 net load 或耦合残差作为目标，风/光因子进入公式具有更明确物理含义（并强调 net load 是定义，发现的是“修正项结构/敏感性规律”）。
