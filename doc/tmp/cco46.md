## 一、最优先：修复数据泄漏问题（影响论文可信度的底线）

文档第4.2节明确指出了两个泄漏源：切分前的缺失值插值（spline/linear + both direction）和气象对齐补值（bfill）。这是目前整个项目可信度的最大隐患，因为一旦审稿人追问"你的流程是否严格无未来信息"，当前无法给出肯定回答。

**具体行动：**

1. **将 `interpolate_missing` 移到切分之后**，对 train/val/test 三段分别独立做插值，且插值方向改为 `limit_direction="forward"` 或只用 `ffill()`，彻底消除未来信息回填。
2. **气象补值同理**，`meteorology.py` 中的 `bfill()` 必须去掉或限制在各段内部执行。
3. **修复后重跑 canonical runs**，至少覆盖：direct `delta_net_load_h6`、`s0p_solar_delta_h6`、`s0p_wind_delta_h6`，以及 matched PySR baseline。如果修复后主要结论不变，论文就可以安全地写"时间因果严格切分"；如果结论发生变化，那反而说明之前的结论确实受到了泄漏污染，早发现比晚发现好。
4. **论文写作的保底策略**：如果时间不够重跑，至少要在 Limitations 中如实披露这一问题，并论证"该泄漏是全局性的、非 wind-specific 的，因此不会系统性地改变 wind vs solar 的对比结论"。

## 二、补充关键实验（强化论文证据链）

### 2.1 多随机种子验证

文档第9节命题F中指出"缺少多随机种子验证"。这是审稿人几乎必问的点。建议对以下核心 run 各跑 3-5 个种子：

- canonical KAN teacher（`delta_net_load_h6`）
- `s0p_solar_delta_h6` 和 `s0p_wind_delta_h6`
- matched PySR baseline

重点观察：公式结构是否跨种子稳定（solar 的物理量是否一直进、wind 是否一直以 lag 形式出现），而不仅仅是 RMSE 数值的方差。

### 2.2 PySR 公平性进一步对齐

文档第5.4节列出了多个未对齐项（训练数据量、归一化空间、搜索强度、operator set）。建议至少补做一个"加强版 PySR run"：增大 `populations` 和 `population_size`，让 PySR 在更充分的搜索预算下运行，看结果是否有质的改变。如果加强后 PySR 仍不如 KAN，论文的 KAN > PySR 结论就更稳。

### 2.3 wind 的消融实验

当前对 wind 的解释是"低符号可辨识性"，但还缺一个直接的消融证据。建议做一组简单实验：在 `s0p_wind` teacher 中，把所有 `wind_lag_*` 从特征列中移除，强制模型只能用原始 `wind_speed_*`。如果此时 raw wind speed 能进入公式但泛化变差，就直接证明了"lag 替代"假说；如果连进入都做不到，则说明问题更深层。

### 2.4 环境依赖补全

文档第12节提到本地缺 `pytest`、`sympy`、`kan` 包。建议尽快在可用环境（比如 Modal 或远程 GPU 机器）上把 `tests.test_physics_mapping` 和 `tests.test_seed_features` 跑通，确保没有被遗漏的 bug。

## 三、论文撰写策略

### 3.1 论文定位与叙事主线

文档最后一段已经给出了非常好的定位：**这是一篇关于"变量可辨识性、替代表达与符号简洁性偏置"的应用研究**，不是模型精度对比稿。论文叙事应围绕以下主线展开：

"我们发现，在净负荷预测任务中，直接做符号提取会退化为负荷主导 → 于是设计了 focused/component teacher → solar 物理量可以稳定进入公式（正例）→ wind 则不行（反例）→ 进一步分析发现 wind 的低可辨识性与 horizon 有非单调关系 → 这说明符号可解释性不仅取决于变量本身的信息量，还受替代表达、时间尺度和符号简洁性偏置的共同影响。"

### 3.2 各章节建议内容

**Introduction**：问题不是"KAN 比 PySR 强多少"，而是"高精度模型学到的知识能否通过符号提取被人类理解？哪些物理因子容易被显式化、哪些不容易？原因是什么？"

**Method**：严格使用文档3.3建议的术语（`KAN teacher with post-hoc symbolic extraction`），避免 distillation 一词。清晰描述 focused teacher 和 component teacher 的设计动机。

**Results**：按文档第10.1节的6条"可以写"来组织，每一条都有对应的 canonical run 证据。表格直接用文档5.3的 matched 对比表。

**Discussion**：
- 把文档第10.2节的4条"只能写成讨论"放在这里
- wind 的 horizon 非单调现象作为核心讨论点
- 历史 PySR > KAN 的情况在此处如实交代并解释
- 数据泄漏问题如果未修复，在此处作为 limitation 披露

**Conclusion**：收束到文档2.2的"一句话版本"——变量能否进入符号公式，不等同于它没有预测价值；wind 的低可辨识性是替代、时间尺度和符号偏置共同作用的结果。

### 3.3 写作时的红线

文档已经非常清晰地划出了不能越过的表述边界，写作时务必遵守：不写"全程 KAN > PySR"、不写"严格无泄漏"、不写"wind 天生特殊已被严格证明"、不写"teacher-student distillation"。
