## 1. 文档目的

本文档用于把当前项目中关于 KAN、PySR、symbolic extraction、wind/solar 物理量进入公式、horizon 依赖性、流程可靠性与论文可写性的系统调查结果固定下来，供后续论文撰写直接引用。

本次核查覆盖：

- canonical runs：`paperref_20260306_121725_v2`、`paperref_20260306_fullflow`、`2026-03-01_151000_kan_nobase_nogrid_gpu`、`2026-03-14_084748_eb4bbc69`
- 变量层证据：`feature_importance.csv`、`formula.sympy.txt`、`predictions_test.parquet`、`comparison_table.csv`


## 2. 先给出总判断

### 2.1 当前最稳的主结论

1. 这里没有真正的 teacher-student 蒸馏。更准确的描述是：`KAN teacher + symbolic extraction`，而不是“训练了一个 symbolic student”。
2. `direct delta_net_load_h6` 路线下，公式确实容易被负荷自身主导；在 canonical direct teacher `paperref_20260306_121725_v2__s1_delta_net_load_h6` 上，这一点非常明显。
3. solar 这条线已经有较强证据表明物理量可以稳定进入公式；wind 则更像“有信息，但原始风速较难在简洁公式中稳定显式化”。
4. wind 的 horizon 现象更接近“先增强、后减弱”的非单调行为，而不是简单的“越长越弱”。
5. 当前流程并非严格无泄漏，主要问题出在切分前的缺失值插值与气象补值；因此论文里的机制性表述必须控制强度。
6. 当前主实验设置下，最新 matched `delta_net_load_h6` 结果已经支持 `KAN > PySR`；但历史 run 中确实存在早期 `PySR > KAN`，这一点必须保留并解释。

### 2.2 当前最稳的一句话版本

在当前净负荷预测任务中，变量能否进入最终符号公式，并不等同于它对预测没有价值；wind 更可能受到时间尺度、变量替代性与符号简洁性偏置的共同影响，因此表现出较低的符号可辨识性。


### 2.3 实验过程演化说明（修正版）

这部分用于说明当前项目是如何从“预测性能比较”逐步转向“物理量能否进入公式”的。

1. 项目早期的重点首先是比较不同方法的预测能力，但需要区分“历史探索期”和“当前主实验期”。历史探索期确实存在个别早期 `load` 目标 run 中 `PySR > KAN` 的情况；不过那一批 KAN 仍属于小模型、弱配置、未形成后期 canonical 设置的阶段，不能直接代表当前主线结论。进入后期 canonical matched `delta_net_load_h6` 主实验后，KAN teacher 已经明确优于直接 PySR，因此当前真正的瓶颈不再是“KAN 能不能赢过 PySR”，而是“高精度 teacher 能不能转化成物理上可信的符号公式”。

2. 正是在这个背景下，项目后续路线才从单纯比较预测精度，转向以 `KAN teacher + symbolic extraction` 为核心的解释链路。这里并不是传统意义上的“教师-学生蒸馏训练”，而是先训练高精度 KAN teacher，再对其进行后验符号提取。转向的直接原因不是为了继续刷预测指标，而是观察到 direct 路线下最终公式过度依赖负荷自身，物理量的进入频率、结构显著性和符号可辨识性都偏低，甚至常常完全进不了最终公式。

3. 在后续大量调参与 focused teacher / component teacher 设计之后，可以看到这个问题并不是“整个链路都无法让物理量进公式”。以 solar、GHI、temperature 为代表的物理量，在多条 focused 路线中已经能够较稳定地进入最终公式，说明当前方法在一般意义上具备把物理信息显式化的能力。

4. 真正持续异常的是 wind。即使在多次调参、特征调整和 teacher 设计之后，wind 在不少设置里仍然表现为：要么原始风速项进不去公式，要么只以 `wind_lag_*`、代理变量或替代表达的形式间接出现，而不是以简洁、稳定的原始风速项显式保留。因此，当前更合理的解释不是“训练失败”，而是 wind 在当前任务与时间尺度下具有更低的符号可辨识性。换句话说，wind 的信息可能并非不存在，而是更容易被历史项或其他变量吸收；而符号提取本身又带有简洁性偏置，因此 raw wind term 更容易被抛弃。

5. 后续 long-horizon 实验进一步支持了这种解释。现有证据更适合写成：wind 在不同 horizon 下呈现出较明显的非单调行为，中等 horizon 更容易显现出可辨识结构，整体上更接近“先增强、后减弱”；而其他物理量目前并不支持同样强的稳定非单调叙事。更稳妥的说法是，其他物理量在更长 horizon 下整体会变得更不稳定，部分量表现出减弱趋势，但不宜简单概括为严格单调下降。

6. 因此，当前项目的实验过程可以概括为：先确认后期 canonical 设置下 KAN teacher 在预测层面优于直接 PySR；随后发现 direct 公式被负荷主导，解释层存在物理量入式不足的问题；再通过 focused teacher 与 component teacher 的连续实验验证“并不是所有物理量都进不去”，最终把问题收敛到 wind 的低符号可辨识性及其 horizon 依赖性上。这个过程本身就是论文讨论部分的重要材料。


## 3. 结构性事实：这里并不存在真正的 teacher-student 蒸馏

### 3.1 代码层定义

- `src/kan_sr/dataset.py` 中 KAN 的训练标签直接来自真实 `target_col`，没有 teacher 输出参与训练。
- `src/local/kan_symbolic.py` 的 symbolic 阶段是：
  1. 读取 Phase 2 的 KAN checkpoint；
  2. 用训练集样本做激活填充；
  3. 对已学到的边做 `suggest_symbolic / fix_symbolic`；
  4. 导出 `formula.sympy.txt`；
  5. 在 test 上用真实 `y_true` 评估公式。

### 3.2 结论

- 当前架构应写成：`KAN teacher -> symbolic extraction`
- 不应写成：`KAN teacher -> symbolic student distillation`
- `seed_from_symbolic_run` 也不是蒸馏，只是把 KAN-symbolic 的子表达式作为 PySR 的附加输入特征

### 3.3 论文表述建议

建议把“教师-学生架构”改成以下口径之一：

1. `teacher-guided symbolic extraction`
2. `KAN teacher with post-hoc symbolic extraction`
3. `KAN-driven interpretable formula extraction`


## 4. 流程可靠性核查

## 4.1 相对可靠的部分

### 4.1.1 时间切分、lag 与 scaler

以下部分没有发现明显实现错误：

- `chronological_split` 使用 48-step gap
- 每个 split 再单独 `drop_initial_lag_rows`
- `normalize_features` 只在 train 上 fit scaler，再应用到 val/test
- `derive_dataset.py` 中 `net_load_lag_*` 与 `delta_*_h{n}` 是在各自 split 内独立构造的
- `transform/inverse_transform` 按 `feature_names` 显式映射，不依赖裸位置

因此：

- 没看到 `wind` 列索引错位
- 没看到 `wind` 列缩放参数串列
- 没看到 `lag` 跨 split 直接引用训练尾部的问题

### 4.1.2 已实际验证的测试

本地可运行且通过：

- `python3 -m unittest tests.test_meteorology tests.test_derived_features tests.test_scaler_transform`
- `python3 -m unittest tests.test_reconstruct_predictions tests.test_ablation_summary`

环境边界：

- 当前本地环境缺少 `pytest`
- 当前本地环境缺少 `sympy`，因此 `tests.test_physics_mapping`、`tests.test_seed_features` 无法继续验证

## 4.2 明确存在的问题：流程不是严格无泄漏

### 4.2.1 缺失值插值发生在切分前

`modal_jobs/data_pipeline.py` 中先调用 `interpolate_missing` 处理全量序列，再做时间切分。  
而 `src/data/preprocess.py` 的插值使用了：

- `method="spline"` 或 `method="linear"`
- `limit_direction="both"`

这意味着：

- 训练段末尾的缺失值可能被未来时间点的信息补回
- 严格因果意义上，这不是无泄漏流程

### 4.2.2 气象对齐与补值也在切分前发生

`src/data/meteorology.py` 中小时气象被对齐到 5min 网格后，再做：

- `interpolate(method="time")`
- `ffill()`
- `bfill()`

这一过程同样发生在切分前。  
因此如果论文要使用“严格时间因果、严格无未来信息”的强表述，当前流程不满足这个条件。

### 4.2.3 对结论的影响

这类问题目前看起来不是 wind-specific bug，而是全局数据管道边界。  
它不会自动推翻“wind 更难显式化”这一观察，但会降低该观察上升为“强机制发现”的可信度。


## 5. KAN vs PySR：必须区分“历史 run”与“当前主实验”

## 5.1 不能直接写成“KAN 一直优于 PySR”

历史 run 中确实存在 `PySR > KAN`：

- `2026-02-26_045336_77244377`：早期 `load` 目标 PySR，`RMSE=127.60`
- 同时期较好的 KAN run `2026-02-26_055200_958b3949`：`RMSE=1241.46`

所以如果写成“全程 KAN 都优于 PySR”，会和仓库证据冲突。

## 5.2 但历史 `PySR > KAN` 不能当作当前主结论

这批历史 run 的问题是：

1. 对应的是早期 `load` 目标，不是当前论文主线的 `delta_net_load_h6`
2. 当时的 KAN 仍处于早期小模型/弱配置阶段，参数与结构都没有形成后期稳定方案
3. 不能把早期探索期 run 与后期 canonical matched run 混在一起下总结论

因此更准确的写法是：

> 历史探索阶段确实出现过早期 `PySR > KAN` 的 run，但那一批 KAN 仍处于小模型、未充分调参和未形成当前 canonical 配置的阶段；在当前主实验使用的 matched `delta_net_load_h6` 设置下，KAN 已经稳定优于直接 PySR。

## 5.3 当前主实验下的 matched 结果

以 `delta_net_load_h6` 的 matched 设置为准：

| run | 说明 | abs RMSE | abs skill |
|---|---|---:|---:|
| `2026-03-01_151000_kan_nobase_nogrid_gpu` | KAN | 1413.51 | 0.453 |
| `2026-03-01_155600_baseline_mlp_match_kan151000` | matched MLP | 1474.38 | 0.430 |
| `2026-03-14_084748_eb4bbc69` | matched PySR | 约 2070 | 约 0.20 |
| persistence | 持久性基线 | 2585.66 | 0 |

这里最重要的结论不是“PySR 全面输”，而是：

1. 当前 canonical `delta_net_load_h6` 设置下，KAN 明显优于 matched PySR
2. PySR 最优式更偏向极简周期项，如 `hour_cos * (const - is_night)`，没有恢复出完整的风/光/温度结构

## 5.4 公平性边界

即便在 matched 设置下，也只能说“部分公平”：

- `feature_cols` 对齐了
- nominal budget 对齐到了 `niterations=1200`

但仍然没有完全对齐：

- KAN 默认 `max_train_rows=50000`
- 当前成功的 PySR matched run 使用全量 train，但搜索强度是 `populations=2, population_size=20, maxsize=15`
- PySR 默认在原始量纲上搜索，KAN 在归一化空间训练
- KAN 的 symbolic lib 与 PySR operator set 也不一致

因此论文里更稳的说法是：

> 在当前 matched `delta_net_load_h6` 设置下，KAN 的预测效果优于直接 PySR；但二者仍不构成绝对严格的一致预算比较。


## 6. canonical direct 路线为何会被负荷主导

## 6.1 canonical direct teacher 的证据

run：`paperref_20260306_121725_v2__s1_delta_net_load_h6`

其特征列包含：

- `load / wind / solar`
- `wind_speed_* / ghi_* / temp_* / surface_pressure`
- `load_lag_12/24/48`
- `wind_lag_12/24/48`
- `solar_lag_12/24/48`

但训练后 `feature_importance.csv` 的结果几乎退化为：

- 只有 `load` 还保留活跃边
- 其他物理量与 lag 基本都被剪到 0

## 6.2 对应 symbolic 结果

以 `paperref_20260306_121725_v2__sym_strict_r2_0_99_paperref_20260306_121725_v2_s1_delta_net_load_h6` 为例，最终公式退化为：

```text
0.000729298697642881*load - 3.00681659753701
```

并且：

- strict / medium / poly4 三类 sweep
- 不同 `r2_threshold`
- 共 9 个 symbolic run

最终都稳定坍缩成只含 `load` 的极简公式。

## 6.3 这说明什么

这条证据很强，支持以下判断：

1. 你最初观察到的“负荷自身对公式影响过大”是真实存在的
2. 这不是单次偶然，而是 current canonical direct teacher 的稳定结果
3. 这也是后来转向更 focused teacher / component teacher 的主要原因


## 7. 物理量进入公式：solar 与 wind 的证据强度并不一样

## 7.1 solar：证据强，且稳定

canonical runs：

- `paperref_20260306_121725_v2__s0p_solar_delta_h6`
- `paperref_20260306_121725_v2__s3_comp_solar_delta_h6`

教师阶段：

- `s0p_solar_delta_h6` 有大量 `solar_altitude / solar_azimuth / is_night / ghi_* / solar_lag_* / temp_2m_c`
- `s3_comp_solar_delta_h6` 同样保留大量 solar 相关活跃边

公式阶段：

- `s0p` 的 3 个 symbolic run 中，`ghi + solar geometry + solar lag + temp` 的进入频率都是 `3/3`
- `s3_comp` 的 3 个 symbolic run 中，这些组的进入频率同样是 `3/3`

结论：

- solar 已经不只是“能进公式”
- 而是“在 canonical 设定下能稳定进公式”

## 7.2 wind：更复杂，不宜简单写成“训练失败”

canonical runs：

- `paperref_20260306_121725_v2__s0p_wind_delta_h6`
- `paperref_20260306_121725_v2__s3_comp_wind_delta_h6`

### 7.2.1 `s0p_wind_delta_h6`

教师阶段：

- 主要保留的是 `wind_lag_12/24`
- `wind_speed_10m_m_s / wind_speed_10m_m_s_cubed / wind_speed_hub_est` 活跃边都是 `0`
- 反而还保留了部分 `ghi_temp_corr_w_m2`、`temp_2m_c`

公式阶段：

- 3 个 symbolic run 都保留了 `wind_lag`
- 3 个 symbolic run 都没有保留原始 `wind_speed_*`
- 3 个 symbolic run 还混入了 `ghi` 与 `temp`

性能：

- abs skill = `0.289`

这说明：

- wind 不是“完全没信息”
- 但 raw wind speed 在这一路线上没有稳定被显式保留

### 7.2.2 `s3_comp_wind_delta_h6`

教师阶段：

- raw wind / cubic wind / hub wind / wind lag 都有明显活跃边

公式阶段：

- 3 个 symbolic run 中 `wind_speed_10m_m_s`、`wind_speed_10m_m_s_cubed`、`wind_speed_hub_est` 的进入频率都是 `3/3`

但性能：

- abs skill = `-0.080`

这说明：

- 让 raw wind 显式进入公式是可以做到的
- 但“显式进入”并不自动等价于“更好的泛化结果”

## 7.3 对 wind 的最稳判断

当前最稳的说法不是：

> wind 进不去是训练失败

而是：

> wind 的预测信息在当前任务中更容易以 lag 或替代表达被模型吸收，而不容易以简洁、稳定、泛化良好的原始风速公式显式保留下来。


## 8. horizon 依赖性：wind 支持“先增后降”，solar 不支持简单单调叙事

## 8.1 wind long-horizon 结果

基于 `exp_longh_wind_delta_h*` 系列：

| horizon | abs skill | wind_speed_edges |
|---|---:|---:|
| h6 | 0.128 | 11 |
| h72 | 0.588 | 0 |
| h144 | 0.354 | 9 |
| h288 | 0.193 | 0 |
| h576 | 0.203 | 0 |

这个结果支持：

1. wind 的表现不是单调下降
2. 中等 horizon 存在更强的可辨识窗口
3. `wind_speed_edges` 也呈现非单调变化

因此可以写成：

> wind 在不同 horizon 下呈现出非单调的可辨识性变化，中等 horizon 更容易显现出可用结构。

## 8.2 solar long-horizon 结果

基于 `exp_longh_solar_delta_h*` 系列：

| horizon | abs skill | ghi_edges |
|---|---:|---:|
| h6 | 0.515 | 0 |
| h72 | 0.590 | 0 |
| h144 | 0.546 | 0 |
| h288 | -7.628 | 2 |
| h576 | 0.225 | 2 |

这说明：

- solar 的 long-horizon 现象也很复杂
- 但并不支持“其他物理量随 horizon 单调下降”这一强表述

所以：

- `wind 先增后降` 可以作为中等强度结论
- `其他变量逐渐下降` 目前证据不足，不宜写成正式主结论

## 10. 哪些结论可以直接写进论文

## 10.1 可以写

1. 当前系统不是经典蒸馏式 teacher-student，而是 `KAN teacher + symbolic extraction`
2. 在 canonical direct `delta_net_load_h6` teacher 下，最终公式会稳定退化为负荷主导
3. 在 focused component teacher 下，solar 的相关物理量可以稳定进入最终符号公式
4. wind 是否进入公式，不完全等同于 wind 是否具有预测价值
5. wind 在不同 horizon 下表现出非单调的符号可辨识性
6. 当前 matched `delta_net_load_h6` 主实验中，KAN 的表现优于直接 PySR

## 10.2 只能写成“讨论/推测”

1. wind 天生就比其他变量更特殊
2. wind 的现象已经被严格证明来自变量替代机制
3. 其他物理量会随着 horizon 单调衰减
4. 当前实验已经是严格无泄漏、严格公平比较


## 11. 论文建议口径

## 11.1 推荐表述

### 11.1.1 关于方法

> 本文采用 KAN 作为高精度教师模型，并在其上进行后验符号提取，以分析哪些物理量能够稳定进入最终解析关系。

### 11.1.2 关于 wind

> 当前结果表明，wind 在教师模型中并非完全无用，但其信息更容易通过历史项或替代变量被吸收，而不易以简洁且稳定的原始风速项显式保留在最终符号公式中。

### 11.1.3 关于 KAN vs PySR

> 历史探索阶段曾出现早期 `PySR > KAN` 的 run，但那一批 KAN 仍处于小模型、未充分调参阶段；在当前 canonical matched `delta_net_load_h6` 主实验中，KAN 已明显优于直接 PySR。

### 11.1.4 关于 direct 路线失败

> 直接在 net-load 目标上提取单条公式时，模型容易退化为负荷主导的极简关系；因此后续实验转向更 focused 的 component teacher，以提升物理变量在解释层中的可辨识性。


## 12. 本次核查的边界

1. 当前结论主要基于已有 run 工件、公式、feature importance 与本地复算结果
2. 没有在当前环境重新训练 KAN，因为本地缺少 `kan` 包
3. 没有完成完整 `pytest` 回归，因为当前环境缺少 `pytest`
4. 没有完成全部 symbolic 单测，因为当前环境缺少 `sympy`
5. 因此本文档的强结论主要依赖已有 run 证据，而不是重新跑出的新实验


## 13. 最终收束

当前项目最值得保住的主线不是“wind 一定特殊”，也不是“teacher-student 蒸馏成功”，而是：

1. 当前 direct net-load 路线确实存在负荷主导退化
2. focused teacher / component teacher 的确能让真实物理因子进入解释层
3. solar 已经提供了较强的稳定正例
4. wind 提供的是更有研究意味的反例与边界：它可能有价值，但不容易被简单、稳定、泛化良好的符号表达显式保留

从论文定位上看，这更接近：

> 一篇围绕“变量可辨识性、替代表达与符号简洁性偏置”的应用研究，而不是单纯的模型精度对比稿。
